# Professional Research Report Skill

## Overview

本 skill 生成咨询级研究报告（Markdown 格式），覆盖 **市场分析、消费者洞察、品牌战略、财务分析、行业研究、竞争情报、投资尽调、宏观经济分析** 等领域。遵循 McKinsey/BCG 咨询标准。

采用两阶段工作流：
1. **Phase 1 — 分析框架生成**：给定研究主题，产出章节骨架、数据需求、分析逻辑、可视化方案
2. **Phase 2 — 报告生成**：在数据收集完成后，综合所有输入生成最终报告

## Data Authenticity Protocol

**严格规则**：报告中所有数据、图表必须来自提供的 **Data Summary** 或 **External Search Findings**。
- **禁止幻觉**：不得编造、估算、模拟数据。缺失数据标注 "Data not available"。
- **可追溯**：每个主要主张和图表可追溯回输入数据包。

## Fintech 领域专用维度

当分析领域为 **Fintech/金融科技** 时，使用以下专用维度补充通用框架：

### 支付清结算
| 维度 | 指标 | 说明 |
|------|------|------|
| 交易规模 | GMV、TPV、交易量 | 支付处理量级和增速 |
| 渗透率 | 移动支付/数字支付占比 | 用户采用程度 |
| 费率趋势 | take rate、MDR | 定价能力变化 |
| 卡组织竞争 | Visa/MC/银联/本地方案 | 清算层竞争格局 |

### 数字银行
| 维度 | 指标 | 说明 |
|------|------|------|
| 获客成本 | CAC、LTV/CAC | 经济模型健康度 |
| 存款成本 | CoF（资金成本）、NIM | 负债端竞争力 |
| 数字渠道 | 线上渗透率、MAU | 数字化程度 |

### 嵌入式金融
| 维度 | 指标 | 说明 |
|------|------|------|
| API 调用量 | 请求数/月、成功率 | 开发者采用度 |
| 商户集成数 | 活跃商户数 | B 端渗透率 |
| 转化率 | 申请→放款转化 | 用户体验指标 |

### 稳定币/加密支付
| 维度 | 指标 | 说明 |
|------|------|------|
| 市值/流通量 | 总市值、日均交易量 | 市场深度 |
| 合规进度 | 牌照数量、司法辖区 | 监管接受度 |
| 监管信号 | 官方声明、执法行动 | 政策风向标 |

# Phase 1: Analysis Framework Generation

## Purpose

给定研究主题，产出完整的 **分析框架**，作为下游数据收集和最终报告生成的蓝图。

## Inputs

| 输入 | 描述 | 必须 |
|------|------|------|
| 研究主题 | 如 "Gen-Z 护肤市场分析" | 是 |
| 范围/约束 | 地理范围、时间范围、行业细分 | 可选 |
| 特定角度 | 用户想探索的假设或角度 | 可选 |
| 领域 | 市场/金融/行业/品牌/消费/投资等 | 推断 |

## Workflow

### Step 1.1: 理解研究主题

识别核心实体和分析领域，确定自然分析维度：

| 领域 | 典型维度 |
|------|----------|
| 市场分析 | 市场规模、增长趋势、市场细分、增长驱动、竞争格局、消费者画像 |
| 品牌分析 | 品牌定位、市场份额、消费者认知、营销策略、竞品对比 |
| 消费者洞察 | 人口统计、购买行为、决策旅程、痛点、场景分析 |
| 金融分析 | 宏观环境、行业趋势、公司基本面、财务指标、估值、风险评估 |
| 行业研究 | 价值链分析、市场规模、竞争格局、政策环境、技术趋势、进入壁垒 |
| 投资尽调 | 商业模式、财务健康、管理层评估、市场机会、风险因素、退出路径 |
| 竞争情报 | 竞品识别、战略对比、SWOT 分析、差异化定位、市场动态 |

### Step 1.2: 选择分析框架与模型

根据领域选择 **2-4 个专业分析框架**：

#### 战略与环境分析

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| **SWOT** | 优势、劣势、机会、威胁 | 品牌评估、竞争定位、战略规划 |
| **PEST/PESTEL** | 政治、经济、社会、技术（+环境、法律） | 宏观环境扫描、市场进入评估 |
| **Porter's Five Forces** | 供应商议价力、买方议价力、新进入者威胁、替代品威胁、行业竞争 | 行业竞争格局、进入壁垒 |
| **VRIO** | 价值、稀缺性、不可模仿性、组织 | 核心竞争力评估 |

#### 市场与增长分析

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| **STP** | 细分、目标、定位 | 市场细分、目标市场选择、品牌定位 |
| **BCG Matrix** | 明星、现金牛、问题、瘦狗 | 产品组合管理 |
| **TAM-SAM-SOM** | 总可服务/可获得市场 | 市场空间量化 |
| **Technology Adoption Lifecycle** | 创新者→早期采用者→早期多数→晚期多数→落后者 | 新兴技术成熟度评估 |

#### 消费者与行为分析

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| **Consumer Decision Journey** | 认知→考虑→评估→购买→忠诚 | 消费者行为路径、触点优化 |
| **AARRR Funnel** | 获客、激活、留存、收入、推荐 | 用户增长分析 |
| **RFM Model** | 最近一次、频率、金额 | 客户价值细分 |
| **Jobs-to-be-Done** | 用户在特定情境下需要完成的"任务" | 需求洞察、产品创新方向 |

#### 金融与估值分析

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| **DuPont Analysis** | ROE = 净利率 × 资产周转率 × 权益乘数 | 盈利能力分解 |
| **DCF** | 自由现金流折现 | 企业/项目估值 |
| **Comparable Company Analysis** | PE、PB、PS、EV/EBITDA 倍数对比 | 相对估值、同行基准 |

#### 竞争与战略定位

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| **Benchmarking** | KPI 逐项对比 | 竞品差距分析 |
| **Strategic Group Mapping** | 沿两个关键维度聚类竞品 | 竞争格局可视化 |
| **Value Chain Analysis** | 主要活动 + 支持活动价值分解 | 成本优势来源、差异化机会 |
| **Blue Ocean Strategy** | 价值曲线、四步动作框架 | 差异化创新、新市场空间 |

#### 行业与供应链分析

| 框架 | 描述 | 适用场景 |
|------|------|----------|
| **Industry Value Chain** | 上游→中游→下游分解 | 行业结构、利润分布 |
| **Gartner Hype Cycle** | 技术触发→期望膨胀→泡沫破裂→稳步爬升→成熟 | 新兴技术成熟度 |
| **GE-McKinsey Matrix** | 行业吸引力 × 竞争实力 | 业务组合优先级 |

#### 选择原则

1. **领域优先**：基于 Step 1.1 识别的领域，选 2-4 个最相关框架
2. **互补性**：选互补而非重叠的框架（如宏观 PESTEL + 微观 Porter's Five Forces）
3. **深度优于广度**：深入应用 2 个框架优于表面堆砌 6 个
4. **数据可行性**：框架必须能被下游数据收集技能支撑
5. **显式映射**：在章节骨架中标注每个章节使用的框架

### Step 1.3: 设计章节骨架

产出层级章节结构，每个章节包含：

1. **章节标题** — 专业、简洁
2. **分析目标** — 本章旨在揭示什么
3. **分析逻辑** — 使用的推理链或框架（必须引用 Step 1.2 选定的框架）
4. **核心假设** — 待验证或反驳的初步假设

### Step 1.4: 定义每章数据查询需求

每章指定需要收集的**精确数据**：

| 字段 | 描述 |
|------|------|
| 数据指标 | 具体指标（如 "2020-2025 中国护肤市场规模（亿元）"） |
| 数据类型 | 定量/定性/混合 |
| 建议来源 | 行业报告、财报、政府统计、社交媒体、电商、调查数据、新闻 |
| 搜索关键词 | 数据收集代理使用的搜索词 |
| 优先级 | P0（必须）/P1（重要）/P2（补充） |
| 时间范围 | 数据覆盖时间段 |

### Step 1.5: 定义可视化与内容结构

| 字段 | 描述 |
|------|------|
| 图表类型 | 折线图、柱状图、饼图、散点图、雷达图、热力图、桑基图、对比表等 |
| 图表标题 | 描述性标题 |
| 数据映射 | 哪些指标映射到 X/Y 轴或分段 |
| 对比表设计 | 列头和对比维度 |
| 论证结构 | "What → Why → So What" 叙事大纲 |

### Step 1.6: 输出完整分析框架

组装为单个结构化的 **分析框架文档**：

```markdown
# [研究主题] 分析框架

## Research Overview
- **研究主题**: [...]
- **范围**: [地理、时间、行业细分]
- **分析领域**: [市场/金融/行业/品牌/消费/...]
- **核心研究问题**: [1-3 个关键问题]

## Framework Selection
| 章节 | 选用框架 | 应用方式 |
|------|----------|----------|
| ... | ... | ... |

## Chapter Skeleton
### 1. [章节标题]
- **分析目标**: [...]
- **分析逻辑**: [...]
- **核心假设**: [...]
- **数据需求**: (见 Step 1.4)
- **可视化方案**: (见 Step 1.5)
### 2. [章节标题]
...

## Data Collection Task List
[汇总所有 P0/P1 数据需求为结构化任务列表]
```

## Phase 1 Quality Checklist

- [ ] 覆盖该领域所有自然维度
- [ ] 2-4 个专业框架已选定并显式映射到章节
- [ ] 框架互补且不重叠，数据可行
- [ ] 每章有明确的分析目标、分析逻辑、核心假设

# Phase 2: Report Generation

## Purpose

接收完成的 **分析框架** 和 **数据包**，综合为最终咨询级报告。

## Inputs

| 输入 | 描述 | 必须 |
|------|------|------|
| 分析框架 | Phase 1 产出的框架文档 | 是 |
| 数据摘要 | 按章节组织的收集数据 | 是 |
| 图表文件 | 生成的图表图片路径 | 可选 |
| 外部发现 | URL 和摘要（用于引用） | 可选 |

## Workflow

### Step 2.1: 验证输入

确认分析框架含章节骨架、数据需求、可视化方案；数据摘要交叉对照 P0 需求。

### Step 2.2: 映射报告结构

1. **摘要** — 执行摘要，关键发现
2. **引言** — 背景、目标、方法论
3. **主体章节** — 从框架章节骨架映射
4. **结论** — 客观综合
5. **参考文献** — GB/T 7714-2015 格式

### Step 2.3: 生成图表

在写报告前生成所有计划的图表。使用数据摘要中的数据，不得编造或平滑数据。

### Step 2.4: 撰写报告

每个子章节遵循 **"视觉锚点 → 数据对比 → 综合分析"** 流程：

1. **视觉证据**：嵌入图表 `![描述](路径)`
2. **数据对比表**：Markdown 对比表
3. **综合叙事分析**："What → Why → So What"（至少 200 字）

每个子章节必须以强有力的分析段落结尾，综合冲突或强化的数据点，揭示潜在的消费者张力或机会，可选以精炼的 "一句话真理" 结尾（blockquote）。

### Step 2.5: 最终自检

确认报告包含所有部分且顺序正确：摘要→引言→主体章节→结论→参考文献。

## Formatting & Tone Standards

### 咨询语调
- **语调**：McKinsey/BCG — 权威、客观、专业
- **语言**：简体中文，保留知名英文专有名词
- **数字格式**：英文千分位分隔符（`1,000`）
- **强调**：重要观点和关键数据使用 **粗体**

### 标题规范
- **编号**：直接使用标准编号（`1.`、`1.1`）+ 标题
- **禁用前缀**：不使用 "第X章"、"Part"、"Section"
- **可用词**：分析、画像、概览、洞察、评估
- **禁用词**："解码"、"DNA"、"秘诀"、"心智图"、"太阳系"

### 洞察深度（"So What" 链）

每个洞察必须连接 **数据 → 消费者心理 → 战略含义**：

```
❌ 差：女性占 60%。策略：针对女性营销。

✅ 好：女性占比 60%，TGI 高达 180。**这表明**
   购买决策由审美和社交认同驱动，而非纯粹功能需求。
   **因此**，媒体投放应转向视觉主导平台（小红书/Instagram），
   将男性受众视为次要送礼场景。
```

### 参考文献
- **行内**：使用 Markdown 链接
- **参考文献部分**：严格遵循 GB/T 7714-2015

## Notes

- 本 skill 在两阶段多步工作流中运行
- 动态标题：将框架中的主题改写为专业简洁的标题
- 结论部分不含详细建议——建议应放在前面主体章节
- **零幻觉政策**：每项陈述、图表、数字必须有输入数据支持
- **可追溯性**：被要求时必须能指出支持某主张的数据来源具体行

# 保险科技行业基础知识

## 核心定义
利用科技手段改造保险产品设计、销售、理赔全过程的企业和服务。

## 细分赛道

| 细分 | 说明 | 代表玩家 |
|------|------|----------|
| 数字保险公司 | 原生互联网保险公司 | 众安保险、Lemonade |
| 保险经纪平台 | 流量端分销多公司产品 | 蚂蚁保、水滴、慧择 |
| 定价科技 | UBI 基于使用量定价 | Root Insurance |
| 嵌入式保险 | 在电商/支付场景嵌入保险 | 抖音电商运费险 |
| 理赔自动化 | AI自动理赔 | 多家 |

## 产业链

```
保险公司（承保） → 经纪平台（获客） → 用户
                ↓
            再保险公司
```

**关键角色：**
- **承保**：必须持牌，承担风险
- **经纪**：获取流量，销售佣金，不承担风险
- **用户**：最终消费者

**关键指标：**
- GWP (Gross Written Premium)：保费规模
- COR (Combined Operating Ratio)：综合成本率 = 赔付率 + 费用率，越低越好
- 赔付率：赔付支出/保费

## 核心成功要素

1. **流量获取能力**：保险获客成本高，平台流量是最大壁垒
2. **产品设计能力**：细分场景产品创新
3. **核保理赔效率**：AI降本空间大
4. **保险牌照资源**：门槛高

## 中国监管要点

- 保险公司需要保险公司牌照
- 保险经纪需要经纪牌照
- 确保保险公司承担风险，平台不能兜底

## 常见分析误区

1. **混淆经纪和承保**：经纪赚佣金不担风险，承保赚利差担风险
2. **低价获客不可持续**：低价引流后转化率不一定能转化为续保
3. **AI 不是万能的**：定价核保降本，但用户获取还是核心

## 当前主要趋势

- 场景化保险创新加速
- 嵌入式保险成为平台流量变现新方式
- AI 核保理赔持续降本
- 互联网巨头通过经纪方式切入

# 核心玩家优先级判定标准

## 优先级定义

| 优先级 | 判定标准 | 扫描策略 |
|--------|----------|----------|
| ⭐ **最高** | 字节已明确战略重点，必须每月追踪 | 每月必须扫描，任何动态都要深度分析 |
| **高** | 和字节现有业务直接竞争/直接相关，或在战略重点区域 | 必须扫描，重大动态深度分析 |
| **中** | 行业重要玩家，但不直接竞争或不在核心区域 | 扫描主要动态，一般不深度分析 |
| **低** | 行业玩家，对字节当前战略影响小 | 仅在重大事件时记录 |

---

## 优先级判定维度（按权重排序）

| 维度 | 加权得分 | 说明 |
|------|----------|------|
| **是否字节已明确重点跟进** | +3分 | 如 Stripe |
| **和字节现有业务直接相关** | +2分 | 支付/信贷/保险/证券任一 |
| **在战略重点区域** | +2分 | 香港/东南亚 |
| **头部Top5公司（按市值/估值）** | +1分 | 赛道头部必须高优先级 |
| **竞争对手** | +1分 | 直接竞品 |
| **新兴技术领域龙头** | +1分 | Agentic AI/AI+金融 |
| **本月宣布大额融资/并购（≥1亿美元）** | +2分 | 当月重大事件，必须优先分析 |

---

## 总分对应优先级

| 总分 | 优先级 |
|------|--------|
| ≥5分 | ⭐ 最高 |
| 3-4分 | 高 |
| 1-2分 | 中 |
| 0分 | 低 |

## 运行时扫描顺序（按优先级）

1. ⭐ 最高优先级：先扫描（HKMA、Stripe、蚂蚁）
2. 高优先级：其次扫描
3. 中优先级：再次扫描
4. 低优先级：最后扫描，时间/搜索预算不够时可以跳过

---

## 动态调整规则

- 如果搜索预算不足，优先保证高+最高优先级
- 同一优先级内，先扫描战略重点区域（**中国内地 > 香港 > 美国/欧洲 > 东南亚 > 墨西哥/其他**）
- 新兴技术领域（Agentic AI）即使总分不高也要优先扫描
- 当月宣布的大额融资/并购（≥1亿美元）自动提升一级优先级，必须纳入报告

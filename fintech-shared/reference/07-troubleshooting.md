# 防坑指南

> 执行报告生成任务时的常见问题与解决方案。

## 坑 1: WebFetch 失败

**现象**：网站拒绝访问或超时
**解决**：
- 重试 2 次
- 仍失败则降级使用搜索结果摘要（Phase 2.3）
- 记录失败原因，不影响整体流程

## 坑 2: 日期提取错误

**现象**：页面没有明确日期，或日期格式混乱
**解决**：
- 优先从 URL 路径提取（如 /2026/04/10/）
- 其次从 meta 标签提取
- 仍无法确定 → 剔除该事件

## 坑 3: 搜索结果太少

**现象**：WebSearch 返回结果 < 5 条
**解决**：
- 尝试不同关键词变体（base_variants → topic_variants）
- 扩大搜索范围到前后1个月
- 如果仍不足，正常结束并提示

## 坑 4: 假链接/编造 URL

**现象**：URL 404，内容与标题不符
**解决**：
- 每个 URL 必须 WebFetch 验证
- 404 直接剔除
- 内容摘要必须来自页面正文，非搜索描述

## 坑 5: 一级信源遗漏

**现象**：HKMA 发了稳定币新闻但没抓到
**解决**：
- 必须直接访问官网新闻页
- 不只依赖搜索
- 搜索关键词要包含所有常见拼写和变体

## 坑 6: 报告生成失败

**现象**：Markdown/HTML 格式错误
**解决**：
- 使用模板确保格式正确
- 关键部分不能为空（Executive Summary、核心结论等）
- 空数据时显示 "本月无相关动态"

## 坑 7: 文件路径错误

**现象**：找不到输出目录或模板文件
**解决**：
- Phase 0 必须正确创建目录 `~/.claude/fintech-reports/{output,logs,data}`
- 使用绝对路径
- 检查模板文件是否存在

## 坑 8: 空搜索结果导致幻觉

**现象**：WebSearch 返回 0 条结果时，模型倾向于编造事件
**解决**：
- 搜索结果为空时，**必须记录空数组 `[]`，绝不编造**
- 记录 `{"query": "...", "events": [], "status": "no_results"}` 到结果文件
- 如果连续 3+ 个查询无结果，检查关键词是否正确替换了占位符

## 坑 9: 时间戳截断导致混淆

**现象**：将 `2026-04-10 14:30:00` 截断为 `14:30` 丢失日期信息
**解决**：
- 所有时间戳必须保留完整日期：`YYYY-MM-DD` 或 `YYYY-MM-DD HH:MM`
- **禁止**只保留时间部分（HH:MM）
- 在 JSON 中间文件中使用 ISO 8601 格式

## 坑 10: 虚假因果推断

**现象**：两个事件无因果关系但报告中写成因果
**解决**：
- 描述事件关联时，**必须使用 SVO 句型**（主语-谓语-宾语）
- 使用"与此同时"而非"因此"除非有明确的因果证据
- 区分"相关性"和"因果性"：
  → 相关：`A 和 B 同期发生`
  → 因果：`A 导致 B`（必须有证据链支撑）

## 坑 11: 趋势证伪条件过期

**现象**：上月报告的"证伪条件"在本月已触发，但新报告未更新
**解决**：
- Phase 5 趋势判断前，**必须先读 `~/.claude/fintech-reports/data/trend-tracker.json`**
- 检查上期每条趋势的证伪条件是否已触发
- 已触发的证伪条件 → 更新趋势状态为 `Falsified` 或 `Weakened`
- 基于当月最新事实调整预测和证伪条件

## 坑 12: HTML 报告 JS 失效导致 tab 不可点击/图表不渲染

**现象**：报告 tab 切换无效，宏观指标图表/世界地图不显示
**根本原因**：
1. **JS 语法错误**：`</script>` 前出现多余的 `});`，导致整个 script 块解析失败。所有函数（switchTab、initWorldMap 等）全部失效
2. **图表初始化时机**：Chart.js/D3 在 `DOMContentLoaded` 时初始化，但非 active tab 的内容被 `display: none` 隐藏，canvas 和 SVG 元素尺寸为 0，渲染失败
3. **CSS 高度限制**：`.chart-container` 使用 `max-height` 而非 `height`，Chart.js 无法正确计算画布尺寸

**解决**：
- **Phase 7 HTML 生成后，必须验证 `</script>` 前无多余括号**：`grep -n "});" 文件 | tail -5`
- **图表实例必须用全局变量跟踪**：`var macroChartInstance = null;`，每次创建前销毁旧实例
- **tab 切换时重新初始化**：`switchTab()` 和 `jumpToTab()` 切换到 overview tab 时必须调用 `initMacroChart()` 和 `initWorldMap()`
- **CSS 必须用 `height` 而非 `max-height`**：`.chart-container { height: 350px; position: relative; }`
- **模板 `html-template.html` 中的对应代码必须同步更新**，不能只改生成文件

**预防措施**：
- `html-template.html` 已包含正确的 `macroChartInstance` 管理和 `switchTab` 重渲染逻辑
- 生成报告后，检查 `<script>` 块结尾是否干净（只有 `</script>`，无 `});`）
- 打开报告后**必须手动点击所有 tab 验证**，不能只看默认 active 的 overview

## 坑 13: 宏观指标数据来源缺失

**现象**：宏观指标表格中的数值没有标注数据截至时间和来源，读者无法验证
**解决**：
- HTML 模板已包含 `macro-data-note` 区块，必须替换 `{{宏观指标数据来源说明}}`
- 每条数据必须标注：指标名称 + 截至时间 + 数据来源 + 可点击 URL
- 特别关注央行利率数据（Fed/ECB/PBC），必须注明截至月份

# AI-Augmented Personal Management: PARA + GTD + AI

## Overview

Combines the proven PARA method (Tiago Forte) and GTD (David Allen) with AI automation for modern personal productivity. AI handles the mechanical sorting and summarizing so you can focus on thinking and doing.

## When to Use

- Organizing your digital notes and files
- Setting up a new personal knowledge management system
- Streamlining your task management workflow
- AI-assisted weekly/monthly reviews
- Building a "Second Brain"

## Core System 1: PARA Method (Tiago Forte)

Source: Tiago Forte, "The PARA Method"

### Four Top-Level Categories

Everything you have gets sorted into one of these four:

### 1. 🚀 **Projects**

**Definition:** Short-term efforts you're actively working on, with a clear goal and completion date.

**Examples:**
- "Launch gto master discord bot v2"
- "Redesign blog homepage"
- "Vacation to Japan 2025"
- "Learn React hooks"

**Each project gets its own folder.** When the project is done, move it to Archives.

### 2. 🎯 **Areas**

**Definition:** On-going responsibilities you need to maintain, no specific end date.

**Examples:**
- "Work"
- "Health & Fitness"
- "Finances"
- "Relationships"
- "Learning"
- "Hobbies"

**Typically 3-7 areas.** Each area has standards you want to maintain, but no specific completion.

### 3. 📚 **Resources**

**Definition:** Topics you're interested in and want to learn more about, but not actively working on a project or area responsibility.

**Examples:**
- "AI Agents"
- "Photography Tips"
- "Vegetarian Recipes"
- "Investing Resources"
- "Travel Destinations"

**Organized by topic, not by project/area.** When you start a project on a topic, move relevant resources into the project folder.

### 4. 📦 **Archives**

**Definition:** Inactive items from the other three categories.

- Completed projects
- Areas you're no longer maintaining
- Resources you're no longer interested in

**Why archive instead of delete?** You can always search later for reference.

### Key PARA Principle

> **Organize by your projects and goals, NOT by broad subject categories.**

Bad: `./documents/` → `./work/` → `./ai/` → ... (too broad)

Good: `./Projects/launch-gto-bot/` (specific active project)

This ensures you're always working on what matters most right now.

## Core System 2: GTD (Getting Things Done)

Source: David Allen

### 5-Step Workflow

#### 1. 📥 Capture

**What:** Get everything that's on your mind out of your head and into an inbox.

- Ideas, tasks, projects, problems, anything that's occupying attention
- "Your mind is for having ideas, not holding them."
- Keep your capturing tool always accessible (phone app, notebook, notes app)

**AI augmentation:** Use voice-to-text (Superwhisper) to capture quickly, AI cleans up the transcript automatically.

#### 2. 🧹 Clarify

**What:** Process your inbox one item at a time — answer: "What is it, and what's the next action?"

- If it's not actionable → trash → someday/maybe → reference (→ Resources)
- If it is actionable → what's the *next physical action* you need to take?

**AI augmentation:** AI can help you identify the next action, break ambiguous tasks into concrete steps.

#### 3. 🗂️ Organize

**Next actions:** Sort by context (Calls, Errands, At Computer, Home, Waiting For)

- Projects: if it needs more than one action
- Waiting For: things you're waiting on others to do
- Calendar: items with specific time/date
- Next Actions: the next step for each project
- Someday/Maybe: things you might do later

**AI augmentation:** AI can auto-sort into contexts based on content.

#### 4. 🔄 Reflect

- **Weekly Review:** Go through all lists, update status, clean up, get clear for next week
- Check off done items, add new actions, re-prioritize

**AI augmentation:** AI generates a weekly summary of what you completed, what's still pending, identifies patterns in how you spend time.

#### 5. ✅ Execute

**Do** the next highest priority action. Trust your system.

### GTD + AI Synergy

| GTD Step | What AI Does |
|----------|--------------|
| Capture | Voice-to-text transcription, cleaning up messy notes |
| Clarify | Break big vague tasks into concrete actionable next steps |
| Organize | Auto-categorize into contexts/projects |
| Reflect | Summarize progress, identify productivity patterns |
| Execute | Help you prioritize based on your stated goals |

## AI Augmentation Workflow

### Daily Capture → Processing

```
1. You capture ideas/tasks via voice/text
2. AI transcribes (if voice) and cleans up
3. AI suggests PARA category + GTD next action
4. You review and adjust (quick 10 seconds)
5. AI moves to the right place
```

### Weekly Review with AI

**AI does this for you automatically:**

1. List all completed tasks from the week
2. List all pending/incomplete tasks
3. Identify which projects are stuck
4. Suggest 3-5 priorities for next week
5. Summarize key insights/learnings from notes

**You just review and approve.**

## Common PARA Mistakes to Avoid

1. **Too many projects** → Keep 5-10 active projects max. More = nothing gets done.
2. **Confusing Areas with Projects** → Project = clear end date. Area = ongoing responsibility.
3. **Not archiving completed projects** → Clutters your active projects. Archive immediately when done.
4. **Putting everything in Resources** → If you're working on it now, it's a Project or Area.

## Getting Started: 10-Minute Setup

1. Create the four top-level folders: Projects, Areas, Resources, Archives
2. Go through your 5 most important current things → put them in Projects
3. List your 3-7 current life responsibilities → put them in Areas
4. Collect your 3-5 main interests/hobbies → put them in Resources
5. Everything else → Archives (seriously, it's okay)
6. Done! You can add more later as you go.

## Best Practices for AI-Augmented PKM

1. **Keep projects active only when actively working** → 5-10 active max
2. **Move completed projects to Archives immediately**
3. **Review and cleanup weekly** (AI can help with this)
4. **When you start a new project, pull relevant resources from Resources into the project folder**
5. **When you finish a project, archive it, and move any generally useful information back to Resources**

## 🎯 专项实践：早9晚9高效工作 + 健康管理

**适用场景：** 每天工作时间长（9:30-21:30），希望早点下班，同时保持身体健康、劳逸结合

### 核心目标

1. **早点下班** — 通过优先级管理，砍掉低价值工作
2. **劳逸结合** — 强制安排休息和健康活动
3. **身体健康** — 把健康当作项目来管理，不是可选项

### 每日时间块设计（12小时工作制优化版）

```
09:30-12:00  | 上午深度工作块 (2.5h)
12:00-13:00  | 午餐 + 轻度活动 (1h) ⭐ 强制休息
13:00-15:30  | 下午深度工作块 (2.5h)
15:30-16:00  | 下午茶 + 活动 (30min) ⭐ 强制休息
16:00-18:30  | 下午第二工作块 (2.5h)
18:30-19:30  | 晚餐 + 休息 (1h) ⭐ 强制休息
19:30-21:30  | 晚间收尾工作块 (2h)
21:30        | 🏠 目标下班时间

每日总计：工作 9.5h | 休息 2.5h
```

### 每日启动流程（AI 辅助）

**每天早上 9:15（上班前 15 分钟）：**

1. **AI 生成今日计划草案**
   ```
   输入：你的项目列表 + 昨日完成情况
   AI 输出：
   - 今日必须完成的 3 件事（如果只能做 3 件）
   - 建议的 2-5 件次要任务
   - 预估完成时间
   - 识别可能的风险/阻塞
   ```

2. **你快速调整（5 分钟）**
   - 确认或调整优先级
   - 标记哪些可以推迟/取消
   - 确认今日下班目标时间

3. **AI 帮你写入日历/任务管理工具**

### 每日结束流程（AI 辅助）

**每天 21:15（下班前 15 分钟）：**

1. **AI 生成今日总结**
   ```
   - 完成了什么？（庆祝）
   - 没完成什么？为什么？（分析）
   - 明日必须优先处理的事项
   - 今日工作时间统计
   ```

2. **你确认明日计划（5 分钟）**
   - 调整优先级
   - 确认是否可以 21:30 准时下班

3. **AI 记录到日志**，准备明日启动

### 早点下班策略

**1. 每日"必须完成"清单（MIT - Most Important Tasks）**

每天只选 3 件真正重要的事，其他都是锦上添花：

```
今日 MIT（必须完成）：
1. [ ] 完成 X 功能开发（2h）
2. [ ] 回复 Y 客户邮件（30min）
3. [ ] 准备明天的 Z 会议（30min）

如果完成以上 3 件，今日目标达成，可以 21:30 下班 ✅
```

**2. 时间盒（Time Boxing）**

给每个任务设定截止时间，到点就停：

```
14:00-16:00 | 功能开发（2h 时间盒）
→ 16:00 闹钟响，不管做完没做完，先停下评估
→ 如果必须继续，申请额外 30min 时间盒
→ 否则记录剩余工作，明日继续
```

**3. 每日"砍掉"清单**

每天问自己：如果今天只能做一半的事，哪些可以砍掉？

AI 帮你识别：
- 哪些任务可以推迟？
- 哪些可以委托？
- 哪些其实没必要做？

### 关键心态

1. **"完成"比"完美"重要** — 准时下班是目标，不是妥协
2. **健康是项目，不是任务** — 需要管理，需要投入时间
3. **休息是投资，不是浪费** — 休息好才能工作好
4. **AI 是助手，不是老板** — 最终决策权在你

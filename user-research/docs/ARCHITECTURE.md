# 用户研究 Skills 系统架构

## 核心目标

产出的研究洞察和报告必须满足四个标准：
- **准确**：结论有充分证据支撑，置信度明确标注
- **全面**：六维度分析矩阵覆盖画像、场景、需求、痛点、竞品、期望
- **深入**：从 observation 到 insight 到 implication，So-What 链至少三层
- **业务价值**：每个洞察对应可追踪的行动项和成功指标

## 四层架构

```
┌─────────────────────────────────────────────────────────────────┐
│  第一层：理论驱动（Theory Layer）                                │
│  /research-plan — 理论匹配 → 假设生成 → 研究设计矩阵              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第二层：数据分析（Analysis Layer）                              │
│  /research-code — 扎根理论三级编码 + JTBD 框架                   │
│  /research-analyze — 六维度分析矩阵 + MECE + 反事实检验          │
│  /research-insight — Observation-Insight-Implication 提炼        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第三层：质量门禁（Quality Gate）                                │
│  /research-review — 逐条审查 + 红队挑战 + 信息密度评分            │
│  【不达标触发 /research-iterate 回退】                            │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  第四层：产出与迭代（Output & Iteration）                        │
│  /research-report — 高密度报告（渐进式披露 + 置信度标注）         │
│  /research-iterate — 根因定位 + 溯源补链 + 版本对比               │
└─────────────────────────────────────────────────────────────────┘
```

## 理论模型库

### 行为改变与转化
- **Fogg Behavior Model** (B=MAT)：动机 × 能力 × 触发
- **Hook Model**：Trigger → Action → Variable Reward → Investment
- **Stages of Change**：前意向 → 意向 → 准备 → 行动 → 维持

### 需求与决策
- **JTBD (Jobs-to-be-Done)**：用户雇佣产品完成的任务
- **Kano Model**：基本型/期望型/兴奋型/无差异型/反向型需求
- **Prospect Theory**：损失厌恶、参照依赖、概率加权

### 体验与留存
- **Customer Journey Map**：阶段 × 触点 × 情感曲线
- **Peak-End Rule**：体验峰值和结尾决定整体记忆
- **Switching Cost**：经济/程序/关系转移成本

### 社会与扩散
- **Diffusion of Innovations**：创新者/早期采纳者/早期多数/晚期多数/落后者
- **Social Proof**：从众、权威、稀缺

## 六维度分析框架

每个研究主题必须覆盖六个维度，形成结构化矩阵：

| 维度 | 核心问题 | 关键理论 | 输出格式 |
|:---|:---|:---|:---|
| **画像** | 谁在完成任务？差异是什么？ | JTBD、技术采纳生命周期 | Persona 卡片（含任务、痛点、收益） |
| **场景** | 在什么情境下完成？触发因素？ | 情境分析、CJM、四力模型 | 场景故事板 + 触发清单 |
| **需求** | 要完成什么任务？优先级？ | Kano、ERG、马斯洛 | Kano 分类表 + 优先级矩阵 |
| **痛点** | 遇到什么阻碍？严重程度？ | FMEA、认知负荷、摩擦审计 | 痛点优先级（严重×频率×可检测） |
| **竞品** | 现在怎么解决？对比如何？ | 竞争格局、参照点理论 | 感知定位图 + 切换成本分析 |
| **期望** | 期望什么？落差在哪？ | ECT、SERVQUAL | 期望-感知差距矩阵 |

## 质量门禁检查清单

### 证据链完整性
- [ ] 每个结论有 ≥2 个独立数据源支撑
- [ ] 定量发现有关联性证据（相关系数/效应量）
- [ ] 定性发现有原始语录支撑，附受访者编号
- [ ] 存在反例被主动寻找并说明

### 逻辑严谨性
- [ ] 从数据到结论的推理步骤无跳跃
- [ ] 相关性与因果性被正确区分
- [ ] 替代解释被考虑并排除/说明
- [ ] 样本代表性声明准确（不夸大）

### 洞察深度
- [ ] 每个 insight 经过 So-What 链 ≥3 层
- [ ] Observation / Insight / Implication 被明确区分
- [ ] 金字塔结构：结论先行，论据分组支撑

### 信息密度
- [ ] 无空话套话（"众所周知"等）
- [ ] 每段有 ≥1 个数据点或用户语录
- [ ] 删除不改变读者认知的句子

### 业务价值
- [ ] 每个洞察对应可行动的建议
- [ ] 建议有 ICE 评分（Impact × Confidence × Ease）
- [ ] 有可追踪的成功指标

## 迭代机制

```
审阅结果分级：
├── 表面润色（文字/格式）→ 直接修改
├── 结构调整（逻辑/重点）→ 重组章节
├── 证据补强（支撑不足）→ 回退到 code/analyze
└── 方向重构（假设偏差）→ 回退到 plan
```

每次迭代必须输出：
- 问题描述
- 根因定位
- 回退环节
- 具体改动
- 验证方式

5. `research-report.json` — 报告撰写
6. `research-review.json` — 质量审阅
7. `research-iterate.json` — 迭代优化

# Skill: research-iterate

## 描述
基于审阅反馈的迭代优化。支持根因定位、溯源补链、版本对比，可回退到任意环节进行修正，确保质量飞轮持续运转。

## 调用方式
```
/research-iterate --review_feedback="审阅意见" --current_deliverable="当前交付物" --iteration_goal="迭代目标" --previous_iterations="历史迭代"
```

## 参数说明
- **review_feedback**: 审阅意见（来自 research-review 的输出）
- **current_deliverable**: 当前交付物（报告/分析/编码等）
- **iteration_goal**: 迭代目标（可选，如"提升证据强度"、"深化洞察"等）
- **previous_iterations**: 历史迭代记录（可选）

## 输出内容
1. 迭代诊断（问题汇总+根因分析+迭代深度判定）
2. 迭代计划（分阶段：快速修复/证据补强/整合验证）
3. 回退执行指南（根据回退环节提供具体步骤）
4. 迭代日志
5. 迭代终止条件

## Prompt
你是一个研究质量优化专家，擅长诊断问题根因、制定迭代策略、追踪改进效果。你的任务是根据审阅反馈，指导研究工作的迭代优化。

### 迭代方法论

#### 迭代深度分层

| 层级 | 触发条件 | 迭代动作 | 预计时间 |
|:---|:---|:---|:---|
| **表面润色** | 文字表达、格式问题 | 直接修改 | 1-2小时 |
| **结构调整** | 逻辑顺序、重点不突出 | 重组章节 | 2-4小时 |
| **证据补强** | 结论支撑不足 | 回退到编码/分析 | 1-3天 |
| **方向重构** | 核心假设被推翻 | 回退到调研设计 | 1-2周 |

#### 根因定位（5 Whys）

对每个审阅问题，追问根因：
```
问题：结论证据不足
↓ 为什么？→ 缺少定量数据支撑
↓ 为什么？→ 研究设计时未规划定量验证
↓ 为什么？→ 对假设的验证方式考虑不充分
↓ 为什么？→ 理论匹配阶段未生成可验证假设
↓ 为什么？→ 研究设计环节质量不足

根因：研究设计阶段假设生成不充分
回退环节：research-plan
```

#### 溯源补链

确定需要回退的环节后，明确补链路径：
```
当前：报告
↓ 问题：洞察证据不足
↓ 回退到：分析
   └── 补充：交叉分析
   └── 输出：更新分析结果
↓ 前进到：洞察
   └── 补充：重新提炼
   └── 输出：更新洞察结果
↓ 前进到：报告
   └── 修改：更新相关章节
```

### 回退执行指南

- **回退到 research-plan**：重新审视研究问题、调整理论匹配、修正假设矩阵
- **回退到 research-code**：明确补充编码范围、更新编码框架、重新统计频次
- **回退到 research-analyze**：识别补充分析维度、处理反例、更新分析结果
- **回退到 research-insight**：重新审视分析结果、深化So-What链、更新洞察矩阵
- **回退到 research-report**：识别问题段落、重组结构、全文质量检查

### 迭代终止条件
- [ ] 审阅问题全部解决或明确接受为局限
- [ ] 无新问题引入
- [ ] 核心结论证据充分
- [ ] 信息密度达标
- [ ] 时间成本可接受

# Skill: user-research-guided

## 描述
新手友好引导式用户研究全流程。给非用研出身的同事一步步引导，自动串联各环节能力，每个步骤都有清晰说明和示例，帮助完成从选题到报告的完整研究。

## 调用方式
```
/user-research-guided --topic="你的研究主题" --context="业务背景"
```

## 参数说明
- **topic**: 研究主题（如"新用户7天留存低"、"用户不点击首页推荐"）
- **context**: 业务背景（产品类型、用户群体、已知数据）

## 工作流程

1. **第一步**：理解需求 → 生成研究方案（自动调用 `/research-plan`）
2. **第二步**：等待用户收集数据 → 编码表确认 Gate 1（自动调用 `/research-code`）
3. **第三步**：交叉分析 → 识别核心发现（自动调用 `/research-analyze`）
4. **第四步**：洞察提炼 → 深化结论（自动调用 `/research-insight`）
5. **第五步**：发现定级 Gate 2 → 用户确认每个发现的级别
6. **第六步**：生成报告 → 飞书格式导出（自动调用 `/research-report` + `/export-feishu`）
7. **第七步**：独立审核 Gate 3 → 给出质量反馈（自动调用 `/research-review`）

## 设计原则

- **低门槛**：每个步骤都有"是什么"、"需要你做什么"、"示例是什么"
- **不跳过**：不会要求用户懂专业术语，所有概念都有解释
- **可暂停**：每一步完成后等待用户确认再继续
- **质量保障**：Gate 2 发现定级 + Gate 3 独立审核确保输出质量

## 专业实践（对标 GitHub chelseajz/user-research-skill）

- **Zone 模型**：AI 自动执行 vs 用户确认点清晰划分
- **Gate 2 发现定级**：5 种状态（正式发现/方向性/降级/加边界/剔除噪音）
- **置信度标注**：高/中/低三档，保守原则
- **Claim 分类**：Fact/Finding/Insight/Hypothesis/Recommendation
- **8 维度审核**：证据链、逻辑、洞察、密度、业务价值、可信度、传播、交付

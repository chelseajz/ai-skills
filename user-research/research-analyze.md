# Skill: research-analyze

## 描述
交叉分析与关键发现识别。基于编码结果和定量数据，进行多维度交叉分析，找出真正影响用户行为的核心驱动力。

## 调用方式
```
/research-analyze --coded_data="编码结果" --quantitative_data="定量数据" --research_questions="研究问题"
```

## 参数说明
- **coded_data**: 编码结果（用户模式卡片）
- **quantitative_data**: 定量数据（问卷、行为数据等）
- **research_questions**: 研究问题

## 输出内容
1. 关键发现（Top 5-7）
2. 用户转化路径分析
3. 用户分群与特征
4. 核心驱动力排序
5. 意外发现
6. 数据局限

## Prompt
你是一个数据分析师，擅长从杂乱的数据中找出关键规律。你的任务是把编码结果和定量数据结合起来，回答"什么真正影响了用户行为"这个问题。

### 分析原则

#### 避免的问题
- **不要简单堆砌数据**：不要罗列"A组X%，B组Y%"这种无意义的对比
- **不要只做单维度分析**：画像、场景、痛点要交叉起来看
- **不要把相关当因果**：看到数字差异要追问"为什么"

#### 核心方法
1. **交叉分析**：两个维度交叉，找关键差异
2. **漏斗分析**：用户在哪个环节流失
3. **对比分析**：成功用户 vs 失败用户的差异
4. **反例检验**：有没有不符合规律的案例？为什么？

### 必做的交叉分析

1. **用户类型 × 关键行为**：不同用户类型的转化率/留存率差异
2. **使用场景 × 痛点分布**：在什么场景下遇到什么痛点
3. **关键行为 × 流失节点**：做了X行为的用户在哪里流失
4. **痛点 × 竞品选择**：什么痛点导致用户选择竞品

### 关键问题清单
1. 谁是高价值用户？他们有什么共同特征？
2. 用户在哪里卡住？卡点与什么因素相关？
3. 成功用户做对了什么？能否复制给其他人？
4. 流失用户的共同模式？能否提前识别？
5. 数据中的意外发现？与预期不符的是什么？

### 质量检查
- [ ] 每个发现都有定性和定量双重证据
- [ ] 做了至少3组交叉分析
- [ ] 反例被主动寻找并解释
- [ ] 区分了"相关"和"因果"
- [ ] 没有堆砌无意义的数字
- [ ] 核心驱动力有明确的影响力排序

## 功能说明

### 核心特性

1. **Zone 模型人机协作**
   - Zone 1：AI 自动执行（数据整理、编码、图表、报告框架）
   - Zone 2：3 个确认点（Gate 1 编码表、Gate 2 发现定级、Gate 3 质量审核）
   - Zone 3：人类决策（开始范围确认、结束发布签字）

2. **发现定级（Gate 2）**
   - 正式发现：证据强，可支撑报告
   - 方向性信号：模式存在但证据不完整
   - 降级置信度：有坑，与结论一起说明
   - 添加边界：只在特定人群/场景成立
   - 剔除噪音：证据太弱或冲突

3. **新手友好引导**
   - 大白话解释每个环节
   - 分步推进，等待用户确认
   - 自动串联专业 skill

## 新手必读：研究设计避坑指南

### 坑 1：不要问用户"你有没有做过"

用户经常记错或乱答，比如：
- 没看过视频的人会说"看过"（社会期望效应）
- 看过的人会说"没看过"（记忆偏差）

**正确做法**：
- 看行为数据（埋点），而不是问用户
- 如果必须用问卷，用对照组设计（看过 vs 没看过）

---

### 坑 2：因果关系要靠对照组

| 错误思路 | 正确思路 |
|---------|---------|
| "看过视频的用户绑卡率更高，说明视频有效" | 需要对比：看过视频的用户 vs 没看过视频的用户，看转化率是否有差异 |

**对照组设计时机**：
- 评估某个功能/内容的效果时 → 考虑设对照组
- 探索用户需求/痛点时 → 通常不需要对照组

---

### 坑 3：数据优先于问卷

| 数据来源 | 可靠性 | 成本 |
|---------|--------|------|
| 行为数据（埋点） | 最高 | 低 |
| 实验数据（A/B 测试） | 高 | 中 |
| 用户访谈 | 中高 | 高 |
| 问卷调查 | 中 | 低 |

**优先排序**：行为数据 > 实验数据 > 访谈 > 问卷

---

### 坑 4：样本量不够 = 结论不可信

| 目的 | 最少样本量 |
|------|-----------|
| 探索性研究（发现方向） | 5-10 人访谈 |
| 定量验证（证明结论） | 100+ 份问卷 |
| 细分人群对比 | 每组 30+ 样本 |

---

### 坑 5：开放式问题太多 = 数据没法分析

| 问题类型 | 适用场景 | 分析难度 |
|---------|---------|---------|
| 开放式（填空） | 探索未知、收集建议 | 高（需人工编码） |
| 量表（1-5 分） | 满意度、程度打分 | 低 |
| 选择题 | 行为、偏好、人口属性 | 低 |

**建议**：少量开放式 + 大量封闭式组合

## 工作流程

1. **研究范围确认** → 确定目的、数据来源、方法
2. **研究方案生成** → 调用 `/research-plan`
3. **定性数据编码** → 调用 `/research-code`，Gate 1 编码表确认
4. **交叉分析** → 调用 `/research-analyze`
5. **洞察提炼** → 调用 `/research-insight`
6. **发现定级（Gate 2）** → 用户确认每个发现级别
7. **报告生成** → 调用 `/research-report` + `/export-feishu`
8. **质量审核（Gate 3）** → 调用 `/research-review`

## 研究范围确认（第一步必做）

每次研究开始前，必须确认以下内容：

### Q1. 研究目的
| 选项 | 说明 |
|------|------|
| A | 曝光/触达：用户有没有看到 |
| B | 认知效果：用户是否理解 |
| C | 行为转化：是否影响了行为 |
| D | 主观满意度：用户觉得好不好 |

### Q2. 可获取的数据
| 选项 | 说明 |
|------|------|
| A | 埋点/行为数据（播放量、点击、转化等） |
| B | A/B 测试数据（实验组 vs 对照组） |
| C | 用户访谈机会 |
| D | 问卷发放渠道 |

### Q3. 评估方法选择

| 条件 | 推荐方法 |
|------|---------|
| 有行为数据 | 直接分析埋点，不用问卷 |
| 有 A/B 测试 | 对比实验组 vs 对照组 |
| 无数据，只有问卷 | 设计实验组/对照组问卷 |
| 探索性研究 | 用户访谈优先 |

### Q4. 目标样本量
| 选项 | 说明 |
|------|------|
| A | 30-50 份（探索性） |
| B | 100-200 份（验证性） |
| C | 300+ 份（细分分析） |

## 适用场景

- 非用研背景的同事需要做用户研究
- 需要从 0 到 1 完成一次完整研究
- 希望有引导式流程，不遗漏关键步骤
- 需要专业质量保障（Gate 机制）

---

## 与专业模式区别

| 维度 | 专业模式 `/user-research` | 引导模式 `/user-research-guided` |
|------|------------------------|-------------------------------|
| 适用人群 | 用研专业人士 | 非用研背景同事 |
| 交互方式 | 细粒度控制 | 一步步引导 |
| 术语 | 专业术语 | 大白话解释 |
| 确认点 | 可选 | 必须确认 |
| 避坑提醒 | 需要自己注意 | 内置避坑指南 |

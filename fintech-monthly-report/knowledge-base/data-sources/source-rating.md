## 信息多维度评分体系

每条采集到的信息必须进行**五维打分**（每项 0-20 分，总分 0-100 分）：

| 维度 | 分值范围 | 打分标准 |
|------|----------|----------|
| **权威性** | 0-20 | 基于信源等级：一级=20，二级=15，三级=10，其他酌情扣分 |
| **新鲜度** | 0-20 | 报告时间范围内首发=20，每超出7天扣2分，超出30天=0 |
| **深度** | 0-20 | 深度分析/完整披露=20，简讯=10，标题党=0 |
| **话题相关性** | 0-20 | 高度相关=20，部分相关=10，不相关=0 |
| **字节财经相关性** | 0-20 | 直接影响字节业务/战略机会=20，行业间接相关=10，完全无关=0 |

### 评分使用规则

| 总分区间 | 分级 | 处理方式 |
|----------|------|----------|
| ≥80 | **核心信息** | 必须深度分析，放在显著位置 |
| 60-79 | **重要信息** | 正常收录分析 |
| 40-59 | **参考信息** | 仅在完整事件清单中列出 |
| <40 | 噪音 | 过滤，不纳入报告 |

## 搜索规则

1. **月度事件搜索格式**：
   - 中文：`"[领域] [YYYY年MM月] 最新动态 监管"`
   - 英文：`"[domain] [Month] [Year] update regulation"`
   - 示例：`"Stripe new product April 2026"`, `"香港 借贷新规 2026年4月"`

2. **信源优先级使用**：
   - 监管政策变化 → **必须用一级信源**（监管官网）
   - 公司公告/财报 → **必须用一级信源**（官网IR / SEC EDGAR）
   - 行业动态 → **优先用二级信源**（Bloomberg / 财新）
   - 融资/初创动态 → **优先用三级信源**（TechCrunch / Crunchbase）

3. **数据质量标注**：
   - `[精确数据]` → 一级信源
   - `[估算数据]` → 二/三级信源估算
   - `[推测数据]` → AI 推断
   - `[数据缺失: X]` → 搜索无结果

4. **链接要求**：
   - 每条信息必须附链接
   - 无法获取链接时，标注「信源名称 + 链接待补充」
   - **禁止凭空生成 URL**

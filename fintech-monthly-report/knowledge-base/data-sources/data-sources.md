## 使用说明
Phase 2 数据搜集前读取本文件，根据事件类型选择最合适的数据源和搜索策略。

## 数据源分类

### A. 内置工具（直接调用）

#### WebSearch（通用搜索）
- **用途：** 新闻、行业动态、公司公告、监管政策
- **优先使用场景：** 月度扫描搜索当月事件
- **搜索策略：**
  - 同一问题用**中文 + 英文双语各搜一次**（不同结果集）
  - 中文查询优先抓取国内信源（财新、第一财经等）
  - 英文查询优先抓取国际信源（Bloomberg、Reuters 等）
  - 必须加月份限定词：`Agentic payment March 2025` `香港 借贷新规 2025年3月`

#### WebFetch（页面精读）
- **用途：** 精读监管文件、官网公告、完整报道
- **触发时机：** WebSearch 返回关键结果片段不够时，对最重要来源全文提取

## 搜索策略模板

### 月度事件扫描查询模板

```
中文: [领域/赛道] [月份] [年份] 最新动态
中文: [地区] [领域] 新规 [YYYY年MM月]
英文: [领域/赛道] [Month] [Year] update
英文: [region] [domain] new regulation [YYYY]

示例:
  "Agentic payment March 2025 product launch"
  "香港 借贷新规 2025年3月"
  "东南亚 BNPL 监管 2025年3月"
  "Stripe new product March 2025"
```

### 监管政策查询

```
中文: [监管机构] [行业] 管理办法 [YYYY年MM月]
英文: [regulator] [industry] regulation [YYYY]
```

### 投融资查询

```
中文: [赛道] 融资 金额 [YYYY年MM月]
英文: [sector] funding round [YYYY] [Month]
```

## 信息抓取格式规范（强制校验）

### 每条事件必须包含的要素

| 要素 | 要求 | 示例 |
|------|------|------|
| **事件日期** | 必须精确到年月日，不能模糊 | `2026年3月15日` |
| **事件简述** | 一句话客观描述事实，不添加主观解读 | `香港金管局公布新的放债人条例修订咨询文件` |
| **信息来源名称** | 必须写明具体来源 | `香港金管局官方网站` |
| **来源链接** | 必须附上可直接访问的URL，禁止凭空生成 | `https://www.hkma.gov.hk/...` |
| **信源等级** | 必须标注一级/二级/三级 | `一级信源` |
| **综合评分** | 五维评分总分（0-100） | `85分` |

### 强制校验检查清单

搜索完成后，必须逐条检查：

- [ ] 所有事件都有准确日期吗？
- [ ] 所有事件都标注了信源等级吗？
- [ ] 所有事件都计算了综合评分吗？
- [ ] 所有事件都附带了来源链接吗？（无法获取链接时，必须标注「信源名称 + 链接待补充」）
- [ ] 监管政策信息真的来自一级信源（监管官网）吗？
- [ ] 公司公告信息真的来自一级信源（官网IR）吗？
- [ ] 有没有编造不存在的URL？（禁止编造）
- [ ] 数据质量标签正确使用了吗？
- [ ] 是否按实体/话题进行了聚类分组？
- [ ] 同一话题下是否按综合评分降序排列？

如果上述检查不通过，必须补充搜索或修正格式，不能交付。

## 数据质量标注规则（必须严格执行）

| 标签 | 使用场景 | 示例 |
|------|---------|------|
| `[精确数据]` | 来自官方财报、央行/监管机构报告 | `蚂蚁消金贷款余额 X 亿元`[精确数据][^1] |
| `[估算数据]` | 来自第三方研究机构 | `行业规模约 X 亿元`[估算数据][^2] |
| `[推测数据]` | 无直接来源、AI 推断 | `预计市场渗透率约 X%`[推测数据] |
| `[数据存在差异]` | 不同来源数字相互矛盾 | `艾瑞 X 亿 vs IDC Y 亿`[数据存在差异] |
| `[需付费来源, 仅获取摘要]` | 付费墙内容 | `据 Nilson Report`[需付费来源, 仅获取摘要] |
| `[数据缺失: {查询词}]` | 搜索无结果 | `[数据缺失: Agentic payment 融资 March 2025]` |
| `[基于有限公开信息]` | 公司披露极少 | 该章节基于有限公开信息，仅供参考。 |

## 信源访问兼容规则（陌生环境/比赛环境适配）

### 访问风险分级

| 分级 | 信源类型 | 应对策略 |
|------|----------|----------|
| ✅ 安全 | 国内监管官网（央行/HKMA/国家金融监管总局）、国内主流媒体 | 优先直接抓取，必须 WebFetch 验证一级信源 |
| ⚠️ 受限 | 国际监管（SEC/FCA/MAS）、国内付费媒体 | 允许使用 WebSearch 公开摘要，标注来源限制 |
| ❌ 高风险 | 国际付费媒体（Bloomberg/WSJ/FT/The Information）、需要登录的数据平台（Crunchbase/CB Insights） | 仅使用 WebSearch 中已经被其他网站转载的公开摘要，无法获取则跳过，**禁止编造** |

### 标注规范（必须执行）

当无法获取完整内容时，必须在信息后标注：
- `[需付费来源，仅获取公开摘要]` — 付费墙内容，只能看到摘要
- `[网络受限，信息来自第三方转载]` — 原网站无法访问，信息来自转载
- `[数据缺失: 具体原因]` — 完全无法获取，留空标注

### 微信公众号抓取实践总结

| 工具组合 | 抓取成功率 | 适用环境 |
|----------|------------|----------|
| OpenCLI + Browser Bridge | ~90%+ | 安装了 OpenCLI 并且 Chrome 装了 Browser Bridge 扩展 |
| weweRSS RSS 服务 | ~90%+ | 提前部署了 weweRSS 服务并配置了 cookie |
| Playwright | ~60-70% | 有 Playwright 但没有 OpenCLI/weweRSS |
| WebFetch + 搜索转载 + 官网同步 | ~30-40% | 只有基础工具 |

**常用Fintech/AI公众号特点：**
- **机器之心** → 大部分文章同步发布在官网 `https://www.jiqizhixin.com/`，可直接抓取
- **新智源** → 部分内容可在搜索结果找到摘要，部分有官网同步
- **赛博禅心** → 微信独家，需要 OpenCLI/weweRSS/Playwright 才能完整抓取

**抓取顺序：**
`OpenCLI (Browser Bridge) → weweRSS → Playwright → 官网同步 → WebFetch → 搜索转载 → 标注摘要`

**禁止行为：** 不尝试在运行时安装 weweRSS/OpenCLI/第三方工具，不编造内容

## 总搜索次数：约 40-50 次（确保全覆盖无遗漏）
- **搜索优先级顺序**（按字节战略优先级排序，优先保证高优先级覆盖）：
  1. **中国内地**监管政策 + 竞品动态
  2. **香港**监管政策 + 竞品动态
  3. **美国/欧洲**技术趋势 + 国际竞品动态
  4. **东南亚**竞品 + 监管
  5. **墨西哥**及其他新兴市场
- 每个战略区域 × 每个核心赛道至少 2-4 次搜索（中文+英文各一次）
- 找到关键事件后，对高重要性事件可追加 1-2 次 WebFetch 全文验证

**优先级：** 中国内地监管 > 香港监管 > Agentic AI/AI+金融技术进展 > 国际头部竞品战略 > 东南亚竞品 > 其他区域

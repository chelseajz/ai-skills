# 护城河分析框架（Moat Analysis）

用于分析竞品护城河深度，判断竞争格局变化。

## 护城河维度（Fintech 行业）

按重要性排序：

### 1. 网络效应
- **需求端网络效应**：用户越多 → 价值越高
  - 支付网络：卖家多 → 用户更吸引用户 → 更多卖家
  - 例子：PayPal / 支付宝 / 微信支付
- **开发者网络效应**：开发者都用 → 商户更容易接入 → 更多开发者
  - 例子：Stripe / Plaid / Airwallex
- **双边平台网络效应**：双边都增长飞轮

### 2. 数据壁垒
- **交易数据积累 → 更好风控 → 更低坏账率 → 更低定价 → 更大规模 → 更多数据
- 这是支付+信贷一体化模式最强的护城河之一
- 字节自身也在走这条路：电商交易数据 → 更好信贷风控

### 3. 牌照/合规壁垒
- 金融是强监管行业
- 牌照获取需要时间/资本/关系
- 多区域牌照本身就是壁垒
- 例子：中国支付牌照，香港 SVF + 放债人牌照，欧盟 EMI 牌照

### 4. 品牌心智
- **C端：用户信任你把钱放你这
- **B端/开发者：首选你，切换成本高
- 例子：Stripe 在开发者心智第一品牌

### 5. 规模经济
- 固定成本高，可变成本低
- 用户越多 → 单位成本越低
- 例子：跨境支付基础设施，信用卡网络

### 6. 成本优势
- 获客成本优势（字节有流量）
- 技术成本优势

### 7. 切换成本
- B端：开发者集成一次用好，不会轻易换
- C端：钱在你这，用户很少换钱包

## 护城河强度判断标准

| 强度 | 说明 |
|------|------|
| **极强** | 3个以上维度强护城河 |
| **强** | 2个维度强护城河 |
| **中** | 1-2个维度护城河 |
| **弱** | 没有可持续护城河，容易被颠覆 |

## 月度应用：竞品护城河变化

当竞品做出新的战略动作时，分析：
1. 这个动作是**加深了哪个护城河维度？
2. 对竞争格局有什么影响？
3. 对字节竞争地位有什么影响？

## 对字节参考

字节自身护城河：
- **流量+场景**：极强（抖音/TikTok电商交易场景就在那里）
- **数据**：正在构建中（行为+交易数据互补，对风控非常有价值）
- **AI技术**：字节大模型优势，可用于风控/客服/Agentic应用
- **牌照壁垒**：逐步构建中（中国+香港已拿到核心牌照）

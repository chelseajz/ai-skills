> 用于 Phase 6 Markdown 报告生成。执行时引用本文件。

---

## 报告结构要求

必须按以下顺序生成完整章节（**按主题分类组织，便于扫描**）：

### 1. 报告标题和统计信息

```markdown
# 全球金融科技月度报告 {YYYY}年{MM}月（{start_date} - {end_date}）

> 扫描统计：去重后事件 {total_events} 个 | 深度分析（≥80分） {focus_events} 个 | 关注事件（60-79分） {watch_events} 个  
> 一级信源：{primary_count} 个 | 二级信源：{secondary_count} 个 | 生成时间：{current_date}
```

### 2. 核心结论（Executive Summary，3-5条）

```markdown
---

## 核心结论

1. **[结论标题]**：[一句话核心结论]。[对字节战略含义]。
2. **[结论标题]**：[一句话核心结论]。[对字节战略含义]。
3. **[结论标题]**：[一句话核心结论]。[对字节战略含义]。
```

至少 3 条，至多 5 条。每条聚焦一个核心判断，**结论先行**。

## 深度分析格式（每个 ≥80 分事件必须严格遵循）

每个 ≥80 分事件展开时，必须包含以下六个维度：

```
### [{事件标题}]（评分：{XX}）

**基本信息卡**：

| 维度 | 信息 |
|------|------|
| 日期 | {YYYY-MM-DD} |
| 地区 | {地区名称} |
| 信源等级 | 一级/二级/三级/四级 |
| 综合评分 | {XX}/100 |
| 信息来源 | [{来源名称}]({url}) |

---

#### 维度 1: 信号（事实层）

- 用 1-2 句话客观描述事件核心事实
- **必须引用原文 quote**：
  - 如果是**外文事件**：保留英文原文，下方附中文翻译
  - 如果是**中文事件**：直接引用原文

> **原文引用**
>
> "英文原文内容..."
>
> *【译】中文翻译内容...*
>
> — 发言人, 职位, 机构. [来源](url)

- 标注炒作/真实判断：【实质落地】/【明确规划】/【概念阶段】/【炒作信号】
- 如果是概念阶段/炒作信号，必须补充说明：为什么可能只是炒作？

#### 维度 2: 背景脉络 + 护城河分析

- 事件发生的历史脉络：这个趋势/公司/产品之前走到哪一步了？
- 行业竞争格局：直接竞品同期做了什么？横向对比 3-5 家
- 护城河变化判断：该动作加深了哪条护城河？从什么强度变为什么强度？
- 监管环境：哪些监管框架影响这个事件？

#### 维度 3: 影响评估（量化层 + 二阶影响）

- 对行业格局的影响：谁受益？谁受损？必须给出 TAM/TAM-at-risk 数字估算
- **二阶影响推导（强制）**：至少两层：一阶 → 二阶 → 三阶
- 量化评估：必须给出至少一个量化锚点（金额/用户/份额），禁止只说"影响较大"
- **对字节财经各产品的影响（强制）**：每个受影响产品必须分析：
  - 竞争压力：威胁等级（高/中/低）
  - 可借鉴点：该功能是否可以复用到字节产品？复制难度
  - 字节优势：字节在该场景有什么独特优势？
  - 行动建议：短期/中期应该做什么

#### 维度 4: 趋势预判（前瞻层）

- 明确判断方向：加速 / 放缓 / 转向 / 伪趋势
- 未来 3-6 个月具体预测（可验证事件 + 时间点）
- 信心等级：高（>70%）/ 中（40-70%）/ 低（<40%）
- 证伪条件：什么信号出现证明判断错误（必须基于当前事实）

#### 维度 5: 缺口与机会 + 竞争窗口期（洞察层）

- 市场普遍忽略了什么？（主流媒体盲区）
- **竞争窗口期分析（强制）**：
  - 窗口期状态：进入窗口 / 退出窗口
  - 剩余时间估算
  - 什么信号意味着窗口彻底关闭？
- **至少 1 条非共识观点**：
  - 市场共识：...
  - 我的判断：...
  - 验证信号：...

#### 维度 6: 行动建议（落地层）

- **具体可执行动作**：
  - 短期建议（1个月内）：...
  - 中期建议（3个月内）：...
- 优先级：P0（必须做）/ P1（建议做）/ P2（可选）
- 关联字节产品：明确指向抖音支付/放心借/月付/TikTok Shop/抖音保险/香港SVF
```

**原文引用规则强调**：
- 外文：保留原文 + 中文翻译，标注发言人和来源
- 中文：直接引用原文，标注来源
- 如果是数据汇总类事件无直接引言：注明「该事件为数据汇总类事件，无直接引言」

### 金字塔原理（结论先行）

### 数据 → 行为 → 战略链

# HTML 报告输出格式规则

> 用于 Phase 6 HTML 报告生成。执行时引用本文件。

## 占位符替换映射表

读取 `examples/html-template.html` 后，按以下映射替换所有占位符：

### 基础信息

| 占位符 | 替换内容 |
|--------|----------|
| `{{YYYY}}年{{MM}}月` | `{YYYY}年{int(MM)}月`（出现两次：标题和导航栏） |

### 宏观指标（概述 tab 新增模块）

| 占位符 | 替换内容 |
|--------|----------|
| `{{中国宏观指标行}}` | 生成 `<tr>` 行，每行一个指标（中国 6 列：指标/数值/环比/同比/数据质量/来源，含香港地区作为子区域行） |
| `{{美国宏观指标行}}` | 同上（美国 4 列：指标/数值/同比/数据质量/来源） |
| `{{香港宏观指标行}}` | 同上（香港 4 列） |
| `{{宏观图表数据}}` | JSON 对象，格式：`{"labels":[...], "datasets":[{"label":"Fed","data":[...]}, ...]}` |
| `{{宏观趋势解读}}` | 1-3 句话总结整体宏观趋势对行业的影响 |
| `{{宏观指标数据来源说明}}` | 生成 `<li>` 列表，每条说明：指标名称 + 最新数据截至时间 + 数据来源名称 + 可点击 URL |

### 事件与深度分析

| 占位符 | 替换内容 |
|--------|----------|
| `{{事件标题}}` / `{{事件URL}}` / `{{事件描述}}` | 事件卡片字段 |
| `{{评分}}` / `{{类别}}` / `{{来源名称}}` / `{{日期}}` | 事件卡片字段 |
| `{{监管事件标题}}` / `{{监管事件URL}}` | 监管政策动态类别下的事件 |
| `{{投融资事件标题}}` / `{{投融资事件URL}}` | 投融资与并购类别下的事件 |
| `{{产品事件标题}}` / `{{产品事件URL}}` | 新产品/新功能类别下的事件 |
| `{{国内竞品事件标题}}` / `{{国内竞品事件URL}}` | 国内竞品动态类别下的事件 |
| `{{稳定币事件标题}}` / `{{稳定币事件URL}}` | 稳定币/区块链类别下的事件 |
| `{{事件N标题/描述/信号/背景/影响/缺口/行动}}` | 深度分析卡片（N=1,2,3） |
| `{{事件N引用原文}}` | 重点事件直接引用的原文（用于 quote-block） |
| `{{事件N来源名称}}` | 引用原文出处来源 |

### 事件列表按类别分组规则

事件列表**不得平铺展示**，必须按以下 5 个类别分组，每组独立 `event-category-section` 卡片：
1. **监管政策动态**（按区域子分组：中国→香港地区/欧美/东南亚）
2. **投融资与并购动态**（按交易规模从大到小排序）
3. **新产品 / 新功能发布**
4. **国内竞品动态**（支付宝碰一下/微信分付等）
5. **稳定币、区块链与加密支付动态**

每组内事件按评分降序排列。某类别无事件时，该区块显示"本月无相关动态"。

**事件分析卡片布局**：
- 事件描述放在最外层顶部（`event-desc-section`），带左侧彩色边框
- 深度分析部分采用双栏分栏布局（`analysis-split`）：左栏放信号+背景，右栏放影响+缺口+行动
- 引用原文使用 `quote-block` 样式，带引号装饰

### 头部公司追踪（P2 新增）

| 占位符 | 替换内容 |
|--------|----------|
| `{{公司追踪概览}}` | "本月追踪 N 家公司，M 家有重大事件更新" |
| `{{公司N名称}}` | 公司名称（如 Stripe） |
| `{{公司N赛道}}` | 赛道分类（如 支付清结算） |
| `{{公司N业务表现}}` | 生成 `<li>` 列表，每个关键指标一行 |
| `{{公司N趋势方向}}` | 【上升/持平/下降/新高】标签 |
| `{{公司N重大事件}}` | 生成 `<li>` 列表，每个事件一行；如有 ≥80 分事件，生成链接到深度分析卡片 |
| `{{公司N战略要点}}` | 生成 `<li>` 列表，每个战略动向一行 |
| `{{公司N护城河变化}}` | → 加深/持平/侵蚀 + 具体维度变化 |
| `{{公司N字节启示}}` | 对字节战略启示 |

**公司追踪 HTML 格式**：
```html
<div class="company-card">
  <h3 class="company-name">
    <span class="company-track">赛道</span>
    公司名称
  </h3>
  <div class="company-section">
    <h4>业务表现</h4>
    <ul>
      <li><strong>关键指标</strong>：数值 <span class="trend-up/up/down/flat">趋势标签</span></li>
    </ul>
  </div>
  <div class="company-section">
    <h4>本月重大事件</h4>
    <ul>
      <li><a href="事件URL">事件标题</a> — 摘要</li>
    </ul>
  </div>
  <div class="company-section">
    <h4>战略要点</h4>
    <ul>
      <li>战略动向</li>
    </ul>
  </div>
  <div class="company-section">
    <h4>护城河变化 <span class="moat-deepen/moat-flat/moat-erode">→加深/持平/侵蚀</span></h4>
    <p>具体维度变化</p>
  </div>
  <div class="company-section">
    <h4>对字节启示</h4>
    <p>战略启示</p>
  </div>
</div>
```

**筛选规则在 HTML 中的体现**：
- 完整展示的公司：完整 `company-card`
- 缩为简表的公司：生成缩略版 `<div class="company-card-compact">`，仅一行
- 归档到附录的公司：在 `<div class="company-archive">` 中以链接列表呈现

## 行业报告列表 HTML 格式

```html
<div class="report-item">
  <a href="URL" class="report-title-link">标题</a>
  <span class="report-source-badge">机构</span>
  <p class="report-desc">摘要</p>
  <div class="report-meta"><span>发布日期</span>·<span>类型</span></div>
  <div class="report-topics"><span class="report-topic-tag">趋势</span></div>
</div>
```

如无报告，替换为 `<p style="color: var(--color-text-muted); text-align: center; padding: 20px;">本月未找到相关深度报告，请持续关注行业动态。</p>`

## 宏观指标 HTML 格式

每个区域的宏观指标行，按以下格式生成 `<tr>`：

```html
<tr>
  <td>指标名称（如"移动支付交易规模"）</td>
  <td>最新数值</td>
  <td>环比（如有）</td>
  <td>同比（如有）</td>
  <td>数据来源</td>
</tr>
```

中国区域需要 6 列（含环比+同比），美国/香港需要 4 列（仅同比）。

## 宏观指标数据来源说明格式

宏观指标区域底部必须包含数据来源说明区块，格式：

```html
<div class="macro-data-note">
  <strong>数据说明：</strong>
  <ul>
    <li><strong>指标/数据名称</strong>：最新数据截至 YYYY年MM月，来源 <a href="URL" target="_blank" rel="noopener">来源名称</a></li>
    ...
  </ul>
</div>
```

**必须标注的数据项**：
- 央行基准利率（Fed/ECB/PBC）：标注截至月份 + 官方来源 URL
- 表格中宏观指标（移动支付/消费信贷等）：标注截至季度 + 来源
- 稳定币市值等市场数据：标注截至日期 + 报道来源

## 重点事件原文引言格式（score ≥ 80）

**外文事件**：
```html
<div class="quote-block">
  &ldquo;英文原文 quote&rdquo;
  <div class="quote-translation">【译】中文翻译</div>
  <div class="quote-source">来源: <a href="URL">来源名称</a> — 发言人</div>
</div>
```

**中文事件**：
```html
<div class="quote-block quote-chinese">
  &ldquo;中文原文 quote&rdquo;
  <div class="quote-source">来源: <a href="URL">来源名称</a> — 发言人</div>
</div>
```

如无引言，注明"该事件为数据汇总类事件，无直接引言"。

## 语言规则

- 面向用户内容必须使用中文（事件标题、描述、趋势、导航、表格）
- 品牌名和产品名可保留英文（Stripe、Mastercard、USDC、Agentic Commerce 等）
- 外文引言保留原文，附中文翻译
- **专业术语首次出现时必须括号解释**：XRP（Ripple公司发行的加密货币）、BNPL（先买后付）、TAM（可寻址市场总规模）等
